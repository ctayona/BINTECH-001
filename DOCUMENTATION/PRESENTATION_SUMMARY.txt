BinTECH Presentation Summary

Project Overview
BinTECH is an intelligent waste segregation and incentive platform that combines an ESP32-based sorting machine, a Node.js and Express backend, a Supabase database, and a web admin and user portal. The system detects waste materials, sorts them through a motorized mechanism, tracks user points, and supports rewards, reporting, and admin control.

Main Features
1. Waste sorting automation with ESP32 hardware.
2. Material detection for paper, plastic, and metal.
3. LCD display for user feedback and welcome messages.
4. Real-time session synchronization between the device and the backend.
5. User point tracking and rewards redemption.
6. Admin dashboards for monitoring, management, and analytics.
7. Backup export for database snapshots stored in Supabase.
8. Safe restore preview that validates backup data without overwriting anything.
9. Archive history for admin account restoration tracking.
10. Mobile and desktop navigation with live point display.

Main Functions and Modules
Backend controllers
- controllers/wasteSorterController.js handles machine sessions, device session lookup, point transfer, and session completion.
- controllers/dashboardController.js provides user stats, activity overview, transaction history, and points display.
- controllers/rewardsController.js manages rewards listing, creation, update, deletion, and redemption.
- controllers/authController.js handles registration, login, logout, Google login, and profile updates.
- controllers/adminController.js manages admin dashboard data, backup export, restore preview, settings, bin management, collections, schedules, accounts, logs, and analytics.

Frontend templates
- templates/USER_DASHBOARD.HTML shows user stats, activity, and navigation.
- templates/USER_QR_CODE.HTML handles QR transfer flow and nav point updates.
- templates/USER_HISTORY.HTML shows waste history and account points.
- templates/ADMIN_DASHBOARD.html shows admin summary, backup export, and management tools.

ESP32 firmware
- ESP32_WASTE_SORTER_COMPLETE.ino and ESP32-CODE.txt control detection sensors, sorting logic, servo movement, LCD messages, bin checks, and backend synchronization.

Backup and Restore Instructions
Backup export
- The backup download exports a JSON snapshot of the system data.
- The backup is centered on Supabase tables, so the saved data reflects the database state.
- The export includes key tables, metadata, generation time, and warnings.
- This is a manual backup meant for review, archiving, and disaster recovery planning.

Restore preview
- Restore should be done in manual preview mode first.
- The restore preview reads the JSON backup and checks that the structure is valid.
- It lists which tables exist, how many rows each table contains, and any missing table warnings.
- The preview does not insert, update, or delete anything.
- This keeps the process safe because nothing in Supabase is overwritten automatically.
- A full restore should only happen after the preview is reviewed and explicitly approved.

Why Supabase matters
- Supabase is the system of record for application data.
- Storing and restoring from Supabase keeps the backup aligned with the live database.
- That makes the backup usable, consistent, and easy to explain during presentation.

System Security Features
1. Supabase-backed authentication and session checks for logged-in users.
2. Role-based access control for admin routes.
3. Head-role restriction for archive restore actions.
4. Server-side handling of sensitive keys and environment variables.
5. Defensive validation of incoming requests and backup payloads.
6. File upload size and type controls in the server layer.
7. Session-based device synchronization instead of trusting the device alone.
8. Admin archive history that records restore events for auditing.

Hardware Security Features
1. Session lock so one material can win a detection cycle.
2. Detection confirmation window to reduce false sensor triggers.
3. Backend synchronization so the ESP32 uses the server as the source of truth.
4. Fail-closed behavior when no valid session or material is active.
5. LCD feedback that shows the active user and current state.
6. Bin occupancy and verification logic to help prevent unsafe sorting actions.
7. Sensor-based classification that reduces accidental or inconsistent sorting.

Important Behaviors to Discuss
1. The system uses backend data as the source of truth for sessions and points.
2. The ESP32 sends machine state to the server so the dashboard can stay synchronized.
3. The sorting logic uses a detection window to confirm material before sorting.
4. The backup feature exports a JSON snapshot of key tables and metadata from Supabase.
5. The restore preview is manual and safe because it only validates the backup and never overwrites live data.
6. Mobile navigation points were fixed so the displayed points match the account state.
7. The dashboard active state was corrected so the wrong tab is not highlighted on mobile.

Likely Questions and Suggested Answers
1. What does the backup download contain?
Answer: It exports a JSON snapshot of the main Supabase tables plus metadata like generation time, app info, and warnings. It is designed so the data can be reviewed or restored later.

2. How is the restore safe?
Answer: The restore preview does not write anything to the database. It validates the backup structure and shows what would be reviewed, so nothing is overwritten by mistake.

3. Why not restore directly?
Answer: Direct restore can overwrite current live data. A preview step is safer because it lets the admin confirm the contents first.

4. Why should backup data be in Supabase?
Answer: Because Supabase is the system of record for the project. Keeping the backup aligned with Supabase makes the data consistent, traceable, and easier to recover.

5. How are points updated?
Answer: Points are updated through backend sessions and transferred into the account points system after a successful waste sort.

6. How does the ESP32 know what material was detected?
Answer: The detection logic combines sensor triggers inside a confirmation window and then locks onto one material so the classification is stable.

7. What are the system security features?
Answer: Authentication, role-based access control, head-only restore approval, server-side secrets, validation, and audit history.

8. What are the hardware security features?
Answer: Detection locking, confirmation timing, backend session sync, fail-closed behavior, and verification logic that helps prevent false sorting actions.

9. What makes the system practical for real use?
Answer: It combines hardware automation, live synchronization, admin controls, and user incentives in one workflow.

10. What future improvements can be discussed?
Answer: A full restore import wizard, better sensor calibration tools, more detailed analytics, and automated backup verification.

Short Closing Statement
BinTECH is a complete waste segregation platform that connects hardware, database services, and admin tools to track waste sorting, reward users, and support safe Supabase-based backup workflows.
